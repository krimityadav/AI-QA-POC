1. What Is This POC?

AI-QA-POC is a working, file-based proof of concept that shows how Claude AI (an AI assistant by Anthropic) can be used to perform every major phase of software QA — from reading a requirement document to producing a final HTML test report — with minimal human effort.

The goal of this showcase is to demonstrate to the internal QA team that AI is not a future concept. It is a tool you can use today in your daily QA activities to dramatically reduce manual work.

Key Point for the Team: You do not need to know how to code to benefit from AI-assisted QA. The AI reads your requirement document, writes the test cases, writes the automation code, and builds the report. Your job is to provide the requirement and review the output.

Step 2Requirement Analysis — Claude Reads and Breaks Down the Requirement

Who does the work: Claude AI (100%)

Claude reads the requirement doc and performs a structured analysis using the SOP defined in architecture/01-requirement-analysis-sop.md (7-step process, 8 quality gates).

What Claude does automatically:

  • Classifies each requirement by type: FUNC, UI, VAL, SEC, PERF, A11Y, DATA, INT

6. Bug Handling — Detection, Reporting & Ticket Information

When Playwright detects a test failure during execution, a structured process kicks in automatically. Claude AI then produces a standalone bug report file per issue — ready to be copied into any ticket system (Jira, Azure DevOps, Asana, Linear, etc.).

6A — What Happens When a Bug Is Found During Execution

1. Playwright detects the failure — the assertion fails (e.g., element not visible, wrong text, timeout).

2. Evidence is auto-captured immediately:

  • Screenshot of the browser at the exact moment of failure → saved to output/artifacts/
  • Video recording of the full test run → saved to output/artifacts/
  • Playwright trace file (full DOM + network snapshot) → saved to output/artifacts/ on first retry

3. Test is marked FAILED in the JSON results (output/test-results.json) and JUnit XML (output/junit-results.xml).

4. Claude AI analyses the failures after execution and produces:

  • One BUG-NNN-{title}.md file per distinct bug — saved in output/bug-reports/
  • One BUG-SUMMARY.md — executive summary of all bugs found in that run

7. What the AI Model Actually Does

Claude AI (model: claude-sonnet-4-6) acts as a senior QA engineer with full STLC knowledge. It communicates through the Claude Code CLI — a terminal interface that lets Claude read files, write code, run commands, and organise output on your local machine.

How Claude Thinks

Claude is a large language model trained on vast engineering and QA knowledge. When you give it a requirement document, it:

  • Understands the business domain and the software's intended behaviour
  • Applies ISTQB-aligned test design techniques automatically
  • Generates syntactically correct TypeScript code that follows Playwright best practices
  • Follows the project governance rules defined in CLAUDE.md every session — automatically, without being reminded

The CLAUDE.md Governance File

CLAUDE.md at the project root contains 7 rules that Claude follows in every session:

RuleWhat It Enforces
Rule 1Every requirement / execution gets its own dedicated folder with a standard sub-structure
Rule 2Every re-run creates a new timestamped folder — old results never overwritten
Rule 3Every project folder must have a README.md with run command and folder map
Rule 4No files at project root — only framework infrastructure and project folders
Rule 5Shared code stays in src/; project-specific code stays in the project folder
Rule 6Output file standards — where each file type goes and when to create it
Rule 7HTML report format — mandatory sections, colour scheme, file size target (20–30 KB)
Key Point: Claude follows these rules automatically in every session. You do not need to remind it. This is how the project maintains consistency across many executions and team members.
