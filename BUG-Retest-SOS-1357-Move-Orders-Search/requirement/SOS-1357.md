# SOS-1357 – Move Orders: Search Box is Missing

## Bug Summary

When attempting to **Move Orders** to a different patient account, the search fields are missing inside the **Assign Patient** module.

---

## Test Environment

| Field      | Value                          |
|------------|-------------------------------|
| URL        | https://dev.dmerocket.com/    |
| Username   | admin@selectortho.net         |
| Password   | Password123!                  |

---

## Pre-condition

Log in to the application using the credentials above. Ensure the account has access to the **Move Orders** feature.

---

## Execution Test Steps

### Step 1 – Navigate to Patient Listing Screen
- From the dashboard, select the client location as **"Rocket"** to load the relevant patient listing screen.

### Step 2 – Open a Patient's Detail Page
- From the patient listing, open the details page of any patient who has existing orders.

### Step 3 – Access the Move Order Feature
- Navigate to the **Orders** section on the patient detail page.
- Click the **Move Order** button.
- A list of orders associated with the patient will be displayed.

### Step 4 – Select an Order
- Select any order from the list.
- Click the **Next** button to proceed.

### Step 5 – View the Assign Patient Module
- The **Assign Patient** module will open, showing a list of patients to move the selected order to.
- The listing page should display the following column headers:
  - MRN
  - First Name
  - Last Name
  - DOB

### Step 6 – Verify Search Fields are Present
- Confirm the following **search boxes** are visible and accessible above the respective columns:
  - [ ] MRN
  - [ ] First Name
  - [ ] Last Name
  - [ ] DOB

### Step 7 – Test Search Functionality
- Enter values in each search field (individually and in combination).
- Verify that the patient list **filters correctly** based on the input provided.

### Step 8 – Complete the Move
- Select a patient from the filtered results.
- Confirm that the selected order is **successfully moved** to the chosen patient account.

---

## Expected Result

All four search fields — **MRN**, **First Name**, **Last Name**, and **DOB** — should be present and fully functional inside the Assign Patient module, allowing users to search and filter patient accounts.

---

## Current Behavior (Bug Reproduction)

When the Assign Patient module opens, the search boxes for **MRN**, **First Name**, **Last Name**, and **DOB** are **not displayed**, making it impossible to search for a patient account to complete the order move.
