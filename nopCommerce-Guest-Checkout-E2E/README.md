## How to Run

> Run from the project root: `C:\Users\Krimit\Desktop\AI-QA-POC\`

### Run all tests
```bash
npx playwright test \
  --config=nopCommerce-Guest-Checkout-E2E/config/playwright.nopcommerce.config.ts
```

### Run a specific suite
```bash
# Happy path only
npx playwright test nopCommerce-Guest-Checkout-E2E/scripts/e2e/guest-checkout.spec.ts \
  --config=nopCommerce-Guest-Checkout-E2E/config/playwright.nopcommerce.config.ts

# Negative tests only
npx playwright test nopCommerce-Guest-Checkout-E2E/scripts/negative/ \
  --config=nopCommerce-Guest-Checkout-E2E/config/playwright.nopcommerce.config.ts
```

### Open the Playwright HTML report
```bash
npx playwright show-report nopCommerce-Guest-Checkout-E2E/output/playwright-report/latest
```

## What Each Folder Is For

| Folder | Open when you want to… |
|---|---|
| `requirement/` | Understand what the feature does and what needs to be validated |
| `scripts/` | Read or modify test automation code |
| `page-objects/` | See how each page's UI is interacted with in code |
| `fixtures/` | Understand how page objects are injected into test functions |
| `test-data/` | See what inputs the tests use (valid and invalid) |
| `config/` | Change browser, timeout, or output settings |
| `tools/` | Run utility scripts (report generation, Cloudflare bypass) |
| `architecture/` | Understand the framework design and SOPs |
| `output/run-history/` | Review results of a past specific run by timestamp |
| `output/playwright-report/latest/` | Open the most recent interactive test report |
| `output/reports/` | View the HTML dashboard summary |
