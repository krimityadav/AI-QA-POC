🔴 Bug Reproduction (Current Behavior)

When editing an authorization record in the Auth Work Queue, the Auth Request Date field is enabled and allows users to modify the date value. The system accepts and saves the modified date, resulting in alteration of the original system-generated Auth Request Date.

🟢 Expected Behavior

The Auth Request Date field should be read-only (non-editable) when viewing or editing authorization information.

The field should display the original system-generated date recorded when the record entered the Auth Work Queue.
Users should not be able to modify the date.
The field should be visually disabled/greyed out to indicate that it is not editable.
Any attempt to edit the field should be prevented by the application.
