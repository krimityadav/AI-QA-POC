Infrastructure Fix Applied — afterAll Hook Timeout

Root Cause: test.afterAll(fn, { timeout: 180_000 }) — the second-argument timeout option is not honoured by Playwright 1.60 when the global config sets TIMEOUT_MS=30000 (30 s). The hook always ran under the 30 s global limit. Cleanup of 5 records takes ~28–35 s, occasionally exceeding this limit.
Secondary cause: The default playwright.config.ts reads RETRY_COUNT=2 from .env, giving Playwright 2 automatic retries. When the afterAll hook timed out, the entire serial block restarted from TC-001 (twice). All 17 TCs passed in each cycle — only the hook failed.
Fix applied to spec (effective for all future runs): 
test.afterAll(async () => {
  test.setTimeout(0);   // ← NEW: removes the hook timeout entirely
  for (const name of createdNames) {
    await cleanupRecordByName(name);
  }
  await browserCtx.close();
}, { timeout: 180_000 });   // ← kept as belt-and-suspenders
test.setTimeout(0) inside a hook body is the Playwright-documented way to remove the timeout for the current hook invocation. Setting it to 0 means "no timeout."
Recommended run command going forward: 
npx playwright test tests/insurance-retest/insurance-name-truncation.spec.ts \
  --project=chromium --retries=0 --reporter=html,line
Adding --retries=0 overrides RETRY_COUNT=2 so a hook issue (if any) does not trigger an unwanted re-run of all 17 TCs.
